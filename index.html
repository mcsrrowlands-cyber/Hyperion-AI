Additional Information
About Me

My name is Max Rowlands, and I am an undergraduate law student at the University of Exeter. Hyperion AI is a personal project which I built from scratch to demonstrate my ability to compile, structure, and deploy AI-powered outputs within the legal and financial space.

How This Was Built

Hyperion AI was developed using the Anthropic Claude API, with a custom scoring engine and a hand-curated dataset covering 30 European jurisdictions. Each jurisdiction's data (spanning permitting timelines, support mechanisms, tax incentives, sovereign risk, and PPA enforceability) was individually researched and structured into a machine-readable format.

The Hyperion AI Agent draws exclusively on primary sources which are publicly available: EU Directives, national energy legislation, Statutory Instruments, and published government guidance. Along with other controls, this minimises the number of hallucinations that Hyperion may produce.

The ethical framework underpinning every output was a core priority which I included in the base document that Claude Code drew from when assisting me collate the data sets. For users, Hyperion AI is governed by a layered compliance architecture aligned with the SRA Standards and Regulations (15/12/24 edition) and the FCA Handbook (08/06/26 edition).

The system is hardcoded to refuse requests that conflict with its ethical framework, to cite only traceable primary sources, and to clearly distinguish between revenue mechanisms such as two-way CfDs, one-way premium tariffs, and merchant exposure - distinctions that are critical to accurate investment analysis but frequently conflated in generic tools. This accuracy is one of the core value propositions provided by the Hyperion AI Agent's energy sector specialism.

The result is a decision-support tool that reflects the rigour expected of legal and financial professionals, built by a student who believes that responsible AI deployment in high-stakes domains must begin with the architecture, not just the disclaimer.

Tools Used

Data & Configuration:

  • JSON files (30 jurisdiction files) - hand-structured data files containing all regulatory, financial, and risk data for each European market.
  • scoring_engine.js - My custom-built scoring algorithm that ranks jurisdictions against the user's profile and technology.
  • system_prompt.txt - My instruction set that tells Hyperion AI how to behave, what rules to follow, and how to structure outputs.
  • CLAUDE.md - My handwritten project rules file that governs how Claude Code behaves when working on this project (workflow, ethical practice, output, etc..)

Core Development:

  • Visual Studio Code - the code editor where all files were written and viewed by myself. Claude Code runs as an extension inside it.
  • Claude Code - Anthropic's AI coding assistant, running inside VS Code. Wrote, edited, and debugged all application code with my oversight - HTML, CSS, JavaScript, and the API layer.

AI & Intelligence Layer:

  • Anthropic Claude API - the AI model powering Hyperion's chat responses. Every user query is routed through this API to generate jurisdiction analysis.
  • Claude Sonnet 4.6 - the specific model being called, defined in api/chat.js.

Version Control & Source Management:

  • Git - tracks every change made to the codebase locally, with a full history of commits.
  • GitHub - hosts the code repository online at mcsrrowlands-cyber/Hyperion-AI. Acts as the source of truth for the project. (anyone can view this.)

Deployment & Hosting:

  • Vercel - hosts and serves the live website. Runs the serverless API functions (api/chat.js, api/health.js) and delivers the static frontend files.
